跟着javidx9大佬教程搓的小玩意，侵删。

CMD Rendering是初版，全部使用基本字符串显示。W/S前后移动，A/D转动视角。

CMD FPS是完善后的FPS原型，加入了纹理采样。WASD四向移动，J/L转动视角，空格发射子弹。

因为CMD本身并不是用于渲染的工具，所以使用起来有很多局限性。而且教程比较老，新版Windows的终端运行对应的程序需要手动修改合适的CMD窗口设置，代码里对窗口大小的设置失效了：

CMD Rendering要求控制台大小为138*46，字号12，行高为1.2，单元格宽度为0.6。

CMD_FPS要求控制台大小为256*144，字号8，行高与单元格宽度均为0.6；


非常不方便，而且弄完项目为了不影响平时使用还要再改回默认。所以只能作为小玩具玩一玩。
